Mixiny a isniejące klasy

.red() {
    color: red;
}
.green-background {
    background-color: green;
}

    .red;
    .green-background();
}
.green-background {
    background-color: green;
}
.my-message {
    color: red;
    background-color: green;
}

Mixiny w przestrzeniach nazw

.theme() {
    .red() {
        color: red;
    }
    .green-background {
        background-color: green;
    }
}
.my-message {
    .theme.red;
    .theme.green-background;
}
.my-message {
    color: red;
    background-color: green;
}

Mixiny z argumentami

.color(@color) {
    color: @color;
}
.background(@color) {
    background-color: @color;
}
.my-message {
    .color(red);
    .background(green);
}
.my-message {
    color: red;
    background-color: green;
}

Mixiny z domyślnymi argumentami

.color(@color: red) {
    color: @color;
}
.background(@color: green) {
    background-color: @color;
}

    .color(pink);
    .background;
}
.my-message {
    color: pink;
    background-color: green;
}

Mixiny - przesilenia

.colors(@color) {
    color: @color;

    Pseudoelementy, pseudoklasy i zapytania @media

    Pseudoelementy

    • Są dynamicznie dodawanie przez przeglądarkę,
    • Pozwalają dodatkowo stylować nasze komponenty,
    • Najbardziej popularne to:
      • ::after
      • ::before
      • ::first-letter
      • ::first-line
      • ::selection

    ::before, ::after

    • Pozwala na dodanie dwóch dodatkowych bloków do danego elementu,
    • Najważniejsza właściwość to 'content'

    Poniżej jest jedynie <div class="hex"></div>:

    ::after a float

    • ::after był często wykorzystywany do naprawiania ostylowanych elementów,
    • Dziś częściej spotyka się flexbox layout do rozmieszczenia elementów,
    • Każdy element wybity z layout’u dzięki float’owi powinien zostać zakończony elementem z clear:both

    ::first-letter, ::first-line, ::selection

    • Pozwala wystylować odpowiednio tekst w zależności od jego położenia lub stanu zaznaczenia,
    • Nie staje się nowym blokiem,
    • Wspiera jedynie podzbiór dostępnych styli

    Pseudoklasy

    • Odpowiadają za stan lub pozycję elementów,
    • Nie dodają żadnych elementów HTML oraz nie wymagają dodania żadnych klas,
    • Pseudo klasy można podzielić na kilka umownych grup dotyczących:
      • Stanu,
      • Pozycji,
      • Formularzy

    Pseudoklasy stanu

    • :active
    • :checked
    • :hover
    • :link
    • :visited
    • :disabled

    Pseudoklasy pozycji

    • :first-child
    • :first-of-type
    • :only-child
    • :only-of-type
    • :last-child
    • :last-of-type
    • :nth-child()
    • :nth-of-type()

    Pseudoklasy formularzy

    • :empty
    • :enabled
    • :focus
    • :invalid
    • :required
    • :optional

    Pseudoselektor :not()

    • Pozwala dodać negację na dany element,
    • Jako argument przyjmuje selektor prosty,
    • Należy uważać na pułapkę hierachii elementów w selektorach bez bezpośredniego zawierania

    Zapytania @media

    • @media [not|only] typ and|or (reguły) { css }
    • Pozwala na określanie kiedy w jakich przypadkach dany styl ma być użyty,
    • Opakowuje poszczególne selektory css lub całe arkusze styli

    @media - typy

    • all
    • print
    • screen
    • speech

    @media - reguły

    • [max-|min-]height
    • [max-|min-]width
    • [max-|min-]aspect-ratio
    • [max-|min-]resolution
    • orientation

    @media - and, or

    • @media screen and (max-width: 499px) { }
    • @media (min-width: 500px) and (max-width: 1365px) { ... }
    • @media (min-width: 1366px), (orientation: landscape) and (min-width: 800px) { ... }

    Flexbox

    • Rozmieszczać elementy pionowo lub poziomo,
    • Elastycznie dostosowuje się do dostępnego miejsca,
    • Pozwala wymusić rozmieszczenie w jędnym wierszu lub kolumnie

    Can I Use (.com table)

    Rozmieszczenie elementów w jednym wierszu

    Tytuł
    X
    .container {
    display: flex;
    flex-flow: row nowrap;

    &__item {
        flex: 0 1 auto;
        &--expandable {
            flex-grow: 1;
        }
        &--expander {
            flex-grow: 5;
        }
    }
}

    Rozmieszczenie elementów w wielu wierszach

    .container {
    display: flex;
    flex-flow: row wrap;

    &__item {
        flex: 0 1 auto;
        &--expandable {
            flex-grow: 1;
        }
    }
}

    Właściwości kontenera

    • display: flex | inline-flex;
    • flex-direction: row | row-reverse | column | column-reverse;
    • flex-wrap: nowrap | wrap | wrap-reverse;
    • flex-flow: flex-direction flex-wrap
    • align-content: flex-start | flex-end | center | space-between | space-around | stretch;
    • align-items: flex-start | flex-end | center | baseline | stretch;
    • justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;

    Właściwości elementów

    • flex-grow: liczba;
    • flex-shrink: liczba;
    • flex-basis: długość | auto;
    • flex: flex-grow flex-shrink flex-basis
    • align-self: auto | flex-start | flex-end | center | baseline | stretch;
    • order: liczba;

    Obliczanie dostępnego miejsca

    • Przeglądarka podczas obliczania wymaganych zmian robi porównuje dostępne miejsce do szerokości wszystkich elementów,
    • flex-basis pozwala manipulować jaka wartość jest wykorzystywana podczas obliczeń

    Flexbox - do czego się nie nadaje

    • Wiele wierszy o różnym rozmieszczeniu elementów,
    • Kontenery z elementami, które mają się rozciągać na kilka wierczy/kolumn,
    • Bonus - proste przypadki gdzie width/height: calc(...) by wystarczył

    Właściwości kontenera

    • display: grid | inline-grid | subgrid;
    • grid-template-columns: [nazwa] szerokość-kolumny [nazwa] szerokość-kolumny ...

    Słowem podsumowania

    Grid i Flexbox nie powinny być używane zamienie.

    Flexbox pozwala na stworzenie layout'u dla elementów w jednym lub więcej wierszy.

    Grid pozwala na stworzenie głównego layout'u strony bez tworzenia problematycznych wrapper'ów.
